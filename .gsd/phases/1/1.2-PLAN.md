# Plan 1.2: Top Navigation Bar (Navbar) Implementation

## Objective
The application currently uses a `Sidebar` which limits horizontal space. As more features (like different document types) are added, a top `Navbar` provides better scalability and frees up horizontal screen real estate for wide data tables (`ttk.Treeview`).
This plan replaces the `Sidebar` with a `Navbar` and adjusts the main layout.

## Context
- `.gsd/ROADMAP.md`
- `src/kontakt/ui/app.py`
- `src/kontakt/ui/views/sidebar.py`
