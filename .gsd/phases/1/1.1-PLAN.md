# Plan 1.1: Database Model Refactor (Invoice -> Document)

## Objective
The current database model uses an `Invoice` class, which is too specific. Budget entities process various documents (Invoices, Bank Statements, Notes).
This plan renames the `Invoice` model to `Document` and adds a `type` field to distinguish between them, laying the groundwork for multi-document support.

## Context
- `.gsd/ROADMAP.md`
- `src/kontakt/database/models.py`
- `src/kontakt/ui/views/invoice_add.py`
- `src/kontakt/ui/views/history.py`
