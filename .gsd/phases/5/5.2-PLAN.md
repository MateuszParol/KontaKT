# Plan 5.2: Modal Selection Windows

## Objective
Improve the user experience for selecting relationships (Contractors, Accounts) within heavily populated databases by replacing drop-downs with searchable modal dialogs.

## Context
- .gsd/ROADMAP.md (Phase 5)
- .gsd/DECISIONS.md
- src/kontakt/ui/views/invoice_add.py

## Tasks

<task type="auto">
  <name>Build Reusable Selection Dialog</name>
  <files>
    src/kontakt/ui/components/selection_modal.py
  </files>
  <action>
    - Create a new file for a `SelectionModal` class inheriting from `ctk.CTkToplevel`.
    - It MUST be fully modal (using `.grab_set()` and `wait_window()`).
    - The window should contain: a search input field (`CTkEntry`), a `ttk.Treeview` showing results, and "Select" / "Cancel" buttons.
    - It should accept a data retrieval callback to populate the tree dynamically based on search.
  </action>
  <verify>grep "grab_set" src/kontakt/ui/components/selection_modal.py</verify>
  <done>A reusable, modal selection component is available.</done>
</task>

<task type="auto">
  <name>Integrate Modals in InvoiceView</name>
  <files>
    src/kontakt/ui/views/invoice_add.py
  </files>
  <action>
    - Replace the `CTkOptionMenu` widgets for Contractor and Accounts with `CTkButton` or non-editable `CTkEntry` widgets.
    - When clicked, open the `SelectionModal` seeded with the respective Peewee queries.
    - Upon selection in the modal, populate the logical state and update the button/entry text with the chosen record's name.
  </action>
  <verify>grep "SelectionModal" src/kontakt/ui/views/invoice_add.py</verify>
  <done>Drop-downs are removed; clicking fields opens a blocking modal to search and select records.</done>
</task>

## Success Criteria
- [ ] Invoice creation utilizes search-ready modals for all related entity selections.
- [ ] Modals block the main UI until a choice is made or canceled.
