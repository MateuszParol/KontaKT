# Plan 4.2: KSeF XML Generator Logic & Export UI

## Objective
Logika `xml.etree.ElementTree` formująca plik FA(2) KSeF wg wymaganych norm Ministerstwa Finansów oraz podpięcie przycisku "Generuj KSeF" obok podglądu faktury.

## Context
- `DECISIONS.md`
- `src/kontakt/ksef/mapper.py` (Nowy)
- `src/kontakt/ui/views/history.py` lub `invoice_creator.py` (do dyskusji gdzie umieścicie pobieranie).

## Tasks

<task type="auto">
  <name>Create FA(2) XML Mapper Service</name>
  <files>
    - src/kontakt/ksef/mapper.py
  </files>
  <action>
    - Stwórz osobną klasę `KsefMapper`, która zaciągnie ID faktury (z modelu `SalesInvoice`).
    - Wyciągnij "Dane Urzędu" z the `Settings` model.
    - Metoda `generate_xml(invoice_id, filepath)` złoży węzły:
      - `Naglowek` (kody formularza FA 2).
      - `Podmiot1` (Sprzedawca - my).
      - `Podmiot2` (Nabywca - relacja `invoice.contractor`).
      - `Fa` (szczegóły: Numery, daty w formacie YYYY-MM-DD, sekcja P_xxx czyli rzędy `SalesInvoiceItem`).
    - Miej na względzie użycie biblioteki standardowej `xml.etree.ElementTree`. Elementy są objęte namespace'ami np. `xmlns="http://crd.gov.pl/wzor/2023/06/29/12648/"`.
  </action>
  <verify>python -c "from kontakt.ksef.mapper import KsefMapper; print('Mapper structure ready')"</verify>
  <done>A robust python module capable of reading `SalesInvoice` and dumping a structured KSeF FA(2) XML exists.</done>
</task>
