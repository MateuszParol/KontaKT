# Plan 1.3: Document View Adjustments

## Objective
Following the database refactor (`Invoice` -> `Document`) and the layout restructuring (`Navbar`), the UI must correctly reflect the concept of generic "Documents" rather than specifically "Invoices". 
This plan updates the UI labels, messages, and logic in `InvoiceAddView` (to be renamed or repurposed conceptually, though for now we can keep the class structure largely similar but with generic text) and `HistoryView` to support `Document`.

## Context
- `.gsd/ROADMAP.md`
- `src/kontakt/ui/app.py`
- `src/kontakt/ui/views/invoice_add.py`
- `src/kontakt/ui/views/history.py`
