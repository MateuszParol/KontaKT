# Plan 5.1: High-Performance Data Lists (Treeview)

## Objective
Replace rendering-heavy custom tkinter scrollable frames with native `ttk.Treeview` widgets to handle arbitrarily large datasets (e.g., thousands of history rows) without UI lag.

## Context
- .gsd/ROADMAP.md (Phase 5)
- .gsd/DECISIONS.md
- src/kontakt/ui/views/accounts.py
- src/kontakt/ui/views/contractors.py
- src/kontakt/ui/views/history.py

## Tasks

<task type="auto">
  <name>Implement tk.Style for Dark Mode Treeview</name>
  <files>
    src/kontakt/ui/app.py
  </files>
  <action>
    - Add a global `ttk.Style()` configuration in the main application setup.
    - Configure the `Treeview` layout to match the application's dark mode (e.g., background `#2b2b2b`, foreground white, selected row highlights).
    - Design it to be easily modifiable for the next milestone's UI overhaul.

<task type="auto">
  <name>Refactor List Views to use Treeview</name>
  <files>
    src/kontakt/ui/views/accounts.py
    src/kontakt/ui/views/contractors.py
    src/kontakt/ui/views/history.py
  </files>
  <action>
    - Remove `CTkScrollableFrame` from `AccountsView`, `ContractorsView`, and `HistoryView`.
    - Introduce `ttk.Treeview` and a right-anchored `ttk.Scrollbar`.
    - Configure columns matching previous display formats (e.g., Symbol/Name for accounts, Number/Date/Amount for History).
    - Remove any hardcoded `.limit(100)` clauses from Peewee queries for these lists.
    - For `HistoryView`, preserve the "Pobierz PK (PDF)" functionality (potentially via capturing Treeview click events or context menus, or adding a button outside that operates on the selected row).
  </action>
  <verify>grep -q "ttk.Treeview" src/kontakt/ui/views/history.py && echo "Matches"</verify>
  <done>Lists render using `ttk.Treeview` and can load the full database without limits.</done>
</task>

## Success Criteria
- [ ] Large lists load instantly.
- [ ] Treeview aesthetic blends reasonably well with the CustomTkinter dark theme.
- [ ] Action buttons (like PDF download) are still accessible (e.g., for the currently selected row).
