## Components

### UI (User Interface)
- **Purpose:** Handles user interaction, rendering views like 'InvoiceAdd', 'History', 'Accounts', 'Contractors'.
- **Location:** `src/kontakt/ui/`
- **Dependencies:** `customtkinter` interactions and delegates to business logic/AI.

### AI Engine
- **Purpose:** Predicts Wn and Ma accounts based on invoice descriptions. Uses Naive Bayes and TF-IDF trained on historical data. Training runs in a background thread to prevent UI blocking.
- **Location:** `src/kontakt/ai/engine.py`
- **Dependencies:** `scikit-learn`, `numpy`

### Database Models
- **Purpose:** Defines the schema and handles data persistence for Accounts, Contractors, and Invoices.

### Configurations
- **Purpose:** Manages application settings and local data paths (AppData).

- **Dependencies:** `platformdirs`

## Data Flow

1. **User Interaction:** User inputs invoice data (e.g., description) in `InvoiceAddView`.
2. **AI Suggestion:** UI requests `ai_engine.predict()` for account suggestions based on the description text.
3. **Save/Persist:** User confirms the invoice, and it is saved via Peewee ORM to the local SQLite database.
4. **Background Training:** Upon app launch, the `AIEngine` fetches historical `Invoice` details from the database and updates its internal prediction model (`MultinomialNB`).
