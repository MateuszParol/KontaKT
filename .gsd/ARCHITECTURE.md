## Data Flow

1. **Initialization:** The application starts in `main.py`, launching the CustomTkinter event loop via `app.py`.
2. **UI Rendering:** `app.py` configures the standard layout (Sidebar + Content Area) and loads a default view (`invoice_add`). 
3. **Navigation:** The Sidebar invokes `show_view()` in `app.py` to swap the displayed view inside the Content Area.
4. **Data Operations:** Views interact with Peewee models (`Account`, `Contractor`, `Invoice`, etc.) defined in `models.py`.
5. **Persistence:** The models use the database connection from `db.py` to read/write data to the local `kontakt.db` file located in the user's OS data directory.
